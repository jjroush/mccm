Build a Claude Code traffic light 🚦

From an unflashed Seeed XIAO ESP32-C6 to a desk status light — machine setup, soldering, flashing, and mccm. For macOS.

You were sent a tiny Seeed XIAO ESP32-C6 microcontroller and a traffic-light LED module (red / yellow / green with a single ground and built-in resistors). By the end of this guide they'll sit on your desk and change color based on what your Claude Code sessions are doing:

LightMeans
GreenClaude is actively working
YellowA session is idle / paused
RedA session needs your input
OffNo live sessions

It plugs into your Mac over USB-C — that single cable both powers the board and carries the status. No batteries, no Wi-Fi, no breadboard.

What we'll do
  1. Gather what you need
  2. Set up your Mac — install the tools
  3. Solder the header pins
  4. Flash the firmware
  5. Wire the traffic light
  6. Set up mccm — the part that watches Claude
  7. Verify & troubleshoot

Budget ~45–60 minutes the first time. The soldering is the only step that needs a careful hand.

1 · What you need

In the package

  • Seeed XIAO ESP32-C6 board
  • A strip of header pins (XIAO ships with them loose — we'll solder them on)
  • Traffic-light LED module (4 pins: R, Y, G, GND)

You'll also need

  • A Mac (this guide is macOS-only)
  • A USB-C data cable — must carry data, not charge-only. If a cable has ever synced a phone or an external drive, it's a data cable.
  • A soldering iron + a little solder (any basic kit; ~$15 is fine)
  • 4 jumper wires to connect the module to the board. Get the type that matches your module's pins — if the module has male pins sticking out, use female-to-female jumpers.
Good news Your module already has resistors built in and a single shared ground, so there's no breadboard and no separate resistors to deal with — just four wires from the module to the board.

2 · Set up your Mac

Goal — install the four tools we need: Apple's compiler tools, the Rust language (the firmware is written in Rust), the flasher (espflash), and mccm itself. Open the Terminal app (⌘-Space, type "Terminal") and paste these one at a time.

2.1 — Apple command-line tools

Provides the C compiler and git. If it's already installed this prints a harmless "already installed" message.

xcode-select --install

2.2 — Homebrew (macOS package manager)

Skip if you already have brew. After it installs, it prints two echo ... >> ~/.zprofile lines + an eval line — run those so brew is on your path.

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

2.3 — Rust

Installs the Rust toolchain via rustup. Accept the default option (just press Enter) when prompted, then load it into your current shell.

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source "$HOME/.cargo/env"

2.4 — espflash (the flashing tool)

cargo install espflash --locked

This compiles a tool and takes a few minutes — that's normal.

2.5 — mccm (watches Claude Code)

brew install jjroush/tap/mccm

2.6 — Get the firmware source

This is the code that runs on the board. Clone it somewhere like your home folder.

git clone https://github.com/jjroush/mccm.git
cd mccm/esp32-firmware
Checkpoint Run cargo --version, espflash --version, and mccm --help. If all three print something (not "command not found"), your Mac is ready. If one fails, close and reopen Terminal so it picks up the new tools, then retry.

3 · Solder the header pins

Why — the XIAO ships with its header pins loose. Until they're soldered, any wire you connect only touches by luck and the lights will flicker or stay dark. Soldering makes the connection permanent and reliable. This is the fiddliest step; take your time.

What you're soldering

Push the header strips into the rows of holes along both long edges of the XIAO (pins pointing down), and solder each pin to its silver ring. You only strictly need four pins for this project — D0, D1, D2 (top three on one side) and GND — but doing all of them keeps the board steady and ready for future projects.

  1. Seat the header strips in the holes. Sitting the board on a breadboard (if you have one) or pressing it flat on the table holds the pins straight while you solder.
  2. Heat the pin and its silver ring together with the iron tip for about a second.
  3. Touch solder to the joint (not the iron tip). It should melt and flow into a small shiny cone. Pull the solder away, then the iron.
  4. Each joint should look like a tiny volcano — shiny and smooth. Dull, blobby, or ball-shaped means reheat it briefly.
  5. Make sure no solder bridges two neighboring pins. If it does, reheat and wipe the excess.
First time soldering? It's very learnable — watch a 3-minute "how to solder header pins" video first. Key rule: heat the metal, feed solder to the metal, never melt solder on the iron and dab it. The board tolerates a few seconds of heat fine; just don't park the iron on one spot for 10+ seconds.

4 · Flash the firmware

Goal — load the program onto the board. You're still in the mccm/esp32-firmware folder from step 2.6.

  1. Plug the XIAO into your Mac with the USB-C data cable. macOS needs no driver — it appears on its own.
  2. Run the flash command: 
    cargo run --release --no-default-features --features board-xiao
  3. Watch for Flashing has completed!, then the program starts. The lights aren't wired yet, so nothing lights up — that's expected. Press Ctrl+C to leave the monitor.
If it says "Timeout while running command" The XIAO sometimes won't auto-enter its flashing mode. Put it in by hand:
  1. Press and hold the BOOT button.
  2. While holding it, unplug and replug the USB-C cable (or tap RESET).
  3. Release BOOT.
  4. Run the same cargo run … command again — it'll connect immediately.

5 · Wire the traffic light

Goal — connect the module's four pins to the board. Match by label, not position, since modules vary. Unplug the board from USB while you wire, to be safe.

Module pinXIAO pad
R (red)D0
Y (yellow)D1
G (green)D2
GND (ground)GND
Polarity Because the module shares one ground, the board turns each light on by sending power out the D0/D1/D2 pad. That matches how the firmware is written, so colors will be correct. If every light is on at once and never changes, your module might be the opposite "common-positive" type — rare, but if so, connect its common pin to 3V3 instead of GND and let me know.

6 · Set up mccm

Goal — connect Claude Code to the light. Two parts: install the hooks that let mccm see your sessions, and run the daemon that pushes status to the board.

6.1 — Install the Claude Code hooks

mccm install

This wires mccm into Claude Code and adds a little status icon to your menu bar. (It only affects Claude Code's settings; nothing invasive.)

6.2 — Run the light daemon

Plug the board back into USB, then run:

mccm led

It finds the board automatically and prints Connected to /dev/cu.usbmodem… followed by a status like state -> G. Leave this running in a Terminal tab — it's the bridge between Claude and the light. (Close it and the light goes dark after ~10 seconds.)

One gotcha to remember The light daemon and the flasher can't use the board at the same time. If you ever re-flash the firmware later, quit mccm led first (Ctrl+C), flash, then start it again.

7 · Verify & troubleshoot

The "it works" sequence

  1. When the board powers up, all three lights flash together three times — a built-in wiring self-test. If you see that, your soldering and wiring are good.
  2. With mccm led running and a Claude Code session active, the light turns green.
  3. Let Claude finish and sit idle — it turns yellow. Ask it something new — back to green.

If something's off

SymptomFix
Flash says "No serial ports could be detected"Charge-only cable. Swap to a data cable. If the board has any light/glow it's getting power, so the cable carries power but not data.
Flash says "Timeout while running command"Hold BOOT, replug USB (or tap RESET), release BOOT, run the flash command again.
Flash says "Device or resource busy"mccm led is holding the port. Quit it (Ctrl+C) and flash again.
Only some lights work, or they flickerA solder joint isn't solid, or a wire isn't seated. Reheat the joint for D0/D1/D2/GND until it's a shiny cone; reseat the jumpers.
Self-test runs but never shows live colorsIs mccm led running? Did mccm install succeed? Start a Claude Code session and watch the daemon print state -> ….
Lights are on but show the wrong colorDouble-check R→D0, Y→D1, G→D2. Swapped wires = swapped colors.

How it all fits together

For the curious — the chain of events when Claude starts working:

Claude Code (hooks fire on start/stop/idle)
  → mccm writes the status to a small file
    → "mccm led" reads it and sends one letter over USB
      → the XIAO lights the matching color

Stuck on any step? Send back exactly what the Terminal printed — the error text is usually enough to pinpoint it.
